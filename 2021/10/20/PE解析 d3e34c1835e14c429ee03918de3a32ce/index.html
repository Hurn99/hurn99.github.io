导入表解析

Hurn
2021-10-20
逆向 Windows

PE解析

导入表

存储着PE程序所需要的API的地址。

看导入表中存的什么。

RVA = 1B1ECh ，010Editor计算出来的FOA=0x81EC ，

实际计算，FOA = RVA - 所在节的虚拟偏移地址地址(VirtualAddress) + 所在节的物理地址(PointToRawData) = 0x1EC + 0x8000 = 0x81EC

文件中0x81EC这个位置并没有东西，也没有API相关的信息，因为这部分是在文件状态是为空的，在程序运行状态才会被填充。

查看导入表程序中运行的状态，方式一，根据PE中的导入表RVA去找内存中真正导入的API函数

程序中的导入表为： ImageBase + RVA = 400000h + 1B1ECh = 41B1ECh

如果值对不上，关闭随机基址

用xdbg查看这个地址：

导入表的结构，5个DWORD元素，一个导入表占20个字节。

第一个导入表（小端序，且都是RVA）：

OriginalFirstThunk = 0001B2E8

TimeDateStamp = 00000000

ForwarderChain = 00000000

Name = 0001B44A
FirstThunk = 0001B098

ImageBase + FirstThunk = 400000 + 1B098 = 41B098，通过xdbg可以看到是MessageBoxW。

这个地址真正存放的0x7767A8A0，也就是user32.MessageBoxW

方式二，在xdbg中查看call 调用，也可以看到call MessageBoxw 实际是call ds:[0x41B098] 也就是第一个导入表中的FirstThunk，这个地址的值为 user32.MessageBoxW

另外补充导入表中的OriginalFirstThunk，实际指向结构体IMAGE_THUNK_DATA

该结构中我们需要关注AddressOfData结构成员，该成员中的数据最高位（红色）如果为1(去掉1)说明是函数的导出序号，而如果最高位为0则说明是一个指向IMAGE_IMPROT_BY_NAME结构(导入表)的RVA（蓝色）地址，此处因为我们找的是导入表所以最高位全部为零。
